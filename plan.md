* o projeto deve ser web

* o projeto deve ter uma grade com 75 casas enumeradas de 1 a 75

* quando clicar em uma casa dessas ela deve ficar de uma cor diferente

* preciso ter um botão na tela para limpar (desclicar) todas as casas

* preciso ter uma opção de "Apresentar", que abre uma segunda tela sem comandos, própria para exibição em datashow ou outra tela externa

* a tela de apresentação deve mostrar apenas o conteúdo necessário para o público, sem os botões de limpar, apresentar ou definir título

## Passo a passo do projeto

1. Definir a estrutura inicial da aplicação web, escolhendo uma página principal para controle do bingo e uma segunda tela para apresentação.

2. Criar a tela principal com uma área para o título do bingo, a grade numerada e os comandos necessários.

3. Criar a grade do bingo com 75 casas, enumeradas de 1 a 75, garantindo que todas fiquem visíveis e organizadas de forma clara na tela.

4. Implementar o comportamento de clique em cada casa da grade, permitindo alternar o estado entre marcada e desmarcada.

5. Aplicar uma cor diferente nas casas marcadas, para que o usuário consiga identificar rapidamente os números já selecionados.

6. Adicionar um controle para definir ou alterar o título exibido no bingo.

7. Adicionar um botão de limpar na tela principal, separado da grade, com uma ação clara para desmarcar todas as casas.

8. Implementar a lógica do botão de limpar, fazendo com que todas as casas voltem ao estado inicial sem marcação.

9. Adicionar a opção "Apresentar" na tela principal, para abrir uma segunda tela ou janela voltada para exibição pública.

10. Criar a tela de apresentação sem comandos administrativos, escondendo botões como limpar, apresentar e definir título.

11. Sincronizar a tela de apresentação com o estado atual do bingo, exibindo o mesmo título, números e marcações da tela principal.

12. Ajustar o layout da tela de apresentação para datashow, priorizando boa leitura à distância e aproveitamento da área da tela.

13. Ajustar o layout da tela principal para funcionar bem em telas maiores e menores, mantendo os números legíveis e os botões fáceis de usar.

14. Testar manualmente os fluxos principais: marcar uma casa, desmarcar uma casa, marcar várias casas, limpar todas as marcações, alterar o título e abrir a tela de apresentação.

15. Revisar a interface final para garantir que o projeto atenda aos requisitos definidos: aplicação web, 75 casas numeradas, mudança de cor ao clicar, botão para limpar tudo e modo de apresentação sem comandos.
